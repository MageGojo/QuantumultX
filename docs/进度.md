## 关键决策
- 单一图标库无法覆盖「AI 专属 + 地区 / 广告 / 兜底」全场景，故 AI 区用 lobehub、其余用 Koolson，各自风格统一。
- 节点筛选用 QuantumultX 的 `server-tag-regex` 在策略组层面完成，免维护节点名单；用**纯排除**而非白名单——白名单只认中英文地区词会漏掉纯 emoji 国旗命名节点，纯排除把 emoji 也纳入排除项，更稳。
- 图标 / 规则 / 解析脚本统一走 `testingcf.jsdelivr.net`（大陆可直连）；放弃 `cdn.jsdelivr.net`（大陆常被 DNS 污染）与配置订阅用 `raw.githubusercontent.com`（大陆被墙→冷启动回环）。
- AI 图标必须走 lobehub 的 `/gh/` 直链（`gh/lobehub/lobe-icons@master/packages/static-png/light/`）而非 `/npm/@lobehub/...`：实测 **QX 图标加载器不认以 `@` 开头的路径段**（`/npm/@scope/`）→ 显示默认 Σ；`/gh/...@master/...`（`@` 在中段，同 Koolson）可正常渲染。（早前为防 gh monorepo 超时改 npm，反而导致 QX 不显示——实测单个小 PNG 的 gh 直链已被 jsDelivr 缓存，不会超时。这是「AI 组显示 Σ 没图标」的根因修复。）
- 方案 B 用 5 组细分 AI（OpenAI/Claude/GoogleAI/Gemini/GoogleSearch）：GoogleAI(AI Studio/Gemini API)、GoogleSearch(搜索)、Gemini(网页) 支持地区不同，blackmatrix7 列表混在一起，故在 `[filter_local]` 手写精确域名拆分。
- QX 策略组不支持单组独立健康检查 URL（官方 sample.conf 实锤），住宅优先 / 故障转移只能用 节点名正则（住宅/家宽/原生/解锁）+ `available` 逼近，无法 1:1 复刻小火箭的 chatgpt.com 探测。
- jsDelivr `@main` 分支链接缓存约 12h，push 后不会立刻刷新（实测 purge 也不能立即生效到边缘）；要立即生效用 commit 锁定链接 `@<sha>`（不可变、秒级），日常自动更新用 `@main`（≤12h 同步）。已写入 README。
- 精简原则：策略组必须 ↔ 分流规则一一对应。删远程规则就同步删对应业务组，避免「定义了但无规则指向」的空挂孤儿组（圈X 面板会显示但永不命中）；未单独分流的流量统一走兜底「漏网之鱼」→ 代理。局域网/私网直连必须保留（否则内网/路由器/NAS 被导去代理失联）。
