# Quantumult X 简约配置 · 设计文档

## 一、需求
1. 订阅解析（机场订阅链接自动解析为节点）。
2. AI 策略组：OpenAI、Claude、GoogleAI（AI Studio / Gemini API）、Gemini（网页）、GoogleSearch（谷歌搜索）。
   - 每个 AI 平台同时支持「自动选择」与「手动选择具体节点」。
   - 自动选择池按各平台**实际支持地区**筛选（详见第四节）：OpenAI / Claude / GoogleAI 排 港澳 + 俄罗斯 + 中国；Gemini 放行港澳；GoogleSearch 仅排中国大陆；并统一排除订阅里的垃圾信息节点（流量 / 到期 / 官网等）。
   - OpenAI / Claude / GoogleAI 额外提供 `AI住宅`（住宅 / 家宽节点优先测速）与 `AI故障转移`（住宅池按序故障转移），逼近小火箭的住宅优先 + 故障转移（QX 限制见第四节）。
   - 自动节点不可用时由 `url-latency-benchmark` 定时重测自动切换。
3. 分流精简：远程规则集仅「广告拦截 + 隐私防护 + 反劫持」（均落「广告拦截」组 reject）；AI 五组、常用业务组（YouTube / TikTok / GitHub / Telegram / 国际媒体 / Spotify）、苹果 / 微软直连、局域网直连、国内（`geoip,cn`）等全部 `[filter_local]` 本地手写；未命中任何规则的流量走兜底「漏网之鱼」→ 代理。
4. 广告拦截、兜底分流（漏网之鱼）。
5. 策略组不带 emoji，但带统一风格图标。
6. 每个分流尽量排除其不支持的国家 / 地区。

## 三、策略组结构
基础组（被业务组引用；除 `AI故障转移` 为 `available` 故障转移外，均为 url-latency-benchmark 自动测速，纯排除正则）：
- `全局自动`：全部节点，排 垃圾 + 中国大陆（回国 / 大陆节点对代理无用）。
- `AI自动`（OpenAI / Claude / GoogleAI 默认）：排 港澳 + 俄罗斯 + 中国，仅取 AI 友好地区。
- `AI住宅`：正向先行只挑 住宅 / 家宽 / 原生 / 解锁 / 流媒体 节点，再排 港澳俄陆 + 垃圾，取延迟最低（住宅 IP 基本能过 OpenAI / Claude 风控）。
- `AI故障转移`（`available`）：在 `AI住宅` 同款住宅池里「按订阅顺序选第一个可用」，稳定 / 粘性，适合 ChatGPT。
- `Gemini自动`（Gemini 网页用）：**放行港澳**（Gemini 2026/3 起支持港），仅排 俄罗斯 + 中国。
- `Google自动`（GoogleSearch 用）：仅排 中国大陆 + 垃圾，含港澳 / 俄罗斯等全部地区。
- `proxy`：static 全部可用节点手动选择池（排 中国大陆 + 垃圾）。
- `代理`：static 手动总组 / 上层入口（`proxy` 手动池 + `全局自动` + `direct`，并内联 `server-tag-regex` 节点）。
- `媒体自动`：正向收 解锁 / 住宅 / 流媒体 / 奈飞 / Disney / HBO 节点 + 排 中国大陆 + 垃圾（放行港台解锁），供「国际媒体」默认测速。

> 注：原「地区自动测速组（香港 / 台湾 / 日本 / 新加坡 / 美国 / 韩国）」已移除；业务组改为依赖 `全局自动 / AI自动` 自动测速 + `代理` 手动选择，免维护一堆地区分组。

业务组（static，AI 主组都追加 `全局自动 / direct` 兜底，订阅缺对应地区节点时不黑洞断网）：
- AI：
  - `OpenAI` / `Claude` / `GoogleAI` = `AI自动`（默认）+ `AI住宅` + `AI故障转移` + `全局自动` + `direct` + 手动池（排 港澳 / 俄 / 中）。
  - `Gemini` = `Gemini自动` + `全局自动` + `direct` + 手动池（放行港澳）。
  - `GoogleSearch` = `Google自动` + `全局自动` + `direct` + 手动池（仅排中国大陆）。
- `YouTube` / `TikTok` / `GitHub` / `Telegram` / `Spotify`：默认 `全局自动`（排 中国大陆 + 垃圾，任意海外池）+ `代理` + `direct` 兜底 + 手动池（Spotify 内容随账号区，可手动切 `direct`）。
- `国际媒体`（Netflix / Disney+ / HBO Max / Prime Video / Hulu）：默认 `媒体自动`（解锁池）+ `代理` + `全局自动` + `direct` 兜底 + 手动池（放行港台解锁）。
- `苹果服务` / `微软服务`：默认 `direct` 直连，可切代理（由 `[filter_local]` 手写域名命中）。
- `国内服务`：默认 `direct`，可切代理（由 `geoip,cn` 命中）。
- `广告拦截`：默认 `reject`。
- `漏网之鱼`：兜底，代理优先（未被任何规则命中的流量走这里）。

> 业务组：`YouTube / TikTok / GitHub / Telegram / 国际媒体` 已找回，各组 `server-tag-regex` 内联节点（与 AI 组同款风格）按「排不支持地区 + 垃圾」筛选，并配 `[filter_local]` 域名规则一一对应；无悬空引用、无孤儿组。`YouTube / TikTok / GitHub / Telegram` 复用 `全局自动` 池（不新增测速组，省电省流量），仅「国际媒体」新增 `媒体自动`（解锁池）。

## 四、各 AI 平台地区差异（2026 查证）与 server-tag-regex
**筛选策略：纯排除（不再用白名单）。** 节点名「不含」任一关键词即入选——白名单只认中英文地区词，会漏掉纯 emoji 国旗（🇯🇵🇺🇸…）命名的节点；纯排除把 emoji 国旗也写进排除项，更稳、零维护。

统一排除垃圾：`剩余|流量|到期|过期|重置|套餐|续费|续期|官网|官方|网址|网站|订阅|客服|工单|公告|访问|失联|防失联|购买|优惠|折扣|有效期|测速|测试|邮箱|更新|频道|群组|电报|欢迎|感谢|游戏|倍率|distance|expire|traffic|reset` 等。

各平台可用地区经查证存在真实差异，故**分别筛选**：

| 地区 | OpenAI | Claude | GoogleAI | Gemini(网页) | GoogleSearch |
|---|---|---|---|---|---|
| 港 / 澳 | ❌ | ❌ | ❌ | ✅(2026/3 起) | ✅ |
| 俄罗斯 | ❌ | ❌ | ❌ | ❌(制裁) | ✅ |
| 美/日/韩/新/台/英/德/法/加/澳洲 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 中国大陆 | ❌ | ❌ | ❌ | ❌ | ❌ |

排除项（含中英文 + emoji 国旗，`\b` 词界避免误伤 CN2 / MO→Mongolia 等）：
- CN（所有代理组都排）：`🇨🇳|大陆|大陸|回国|回國`。
- 港澳：`🇭🇰|香港|Hong ?Kong|\bHK\b|🇲🇴|澳门|澳門|Macau|Macao|\bMO\b`。
- 俄罗斯：`🇷🇺|俄罗斯|俄羅斯|俄国|俄國|莫斯科|Russia|Moscow|\bRU\b`。

各组排除组合：
- `OpenAI / Claude / GoogleAI`（走 `AI自动`）：排 港澳 + 俄罗斯 + 中国 + 垃圾。
- `Gemini`（走 `Gemini自动`）：排 俄罗斯 + 中国 + 垃圾（放行港澳）。
- `GoogleSearch`（走 `Google自动`）：排 中国大陆 + 垃圾（其余全收，含港澳 / 俄罗斯）。
- `全局自动 / 代理 / proxy / YouTube / TikTok / GitHub / Telegram / Spotify`：排 中国大陆 + 垃圾。
- `国际媒体 / 媒体自动`：正向收 解锁 / 住宅 / 流媒体 / 奈飞 / Disney / HBO + 排 中国大陆 + 垃圾（放行港台解锁）。
- **俄罗斯节点归属**：OpenAI / Claude / GoogleAI / Gemini 均封 → 只会进 `GoogleSearch` 与 `全局自动 / 代理`，不误入 AI 四家。

### 住宅优先 / 故障转移（逼近小火箭，受 QX 限制）
小火箭用 `AI-Fallback` 以 `chatgpt.com/cdn-cgi/trace` 做健康检查，能识别被 OpenAI 拉黑的机房 IP。**QX 不支持给单个策略组设独立 `check-url`**（只能用全局 `server_check_url`，仅 per-server 可覆盖），故无法 1:1 复刻。替代方案：
- `AI住宅`（`url-latency-benchmark`）：正向先行 `(?=.*(家宽|住宅|原生|解锁|流媒体|Residential|Native…))` 只挑住宅级节点，这类住宅 IP 基本能过 OpenAI / Claude 风控，取延迟最低。
- `AI故障转移`（`available`）：同款住宅池按订阅顺序选第一个可用（稳定 / 粘性，适合 ChatGPT 会话）；健康检查仍走全局 `server_check_url`（gstatic），只判连通、不判 OpenAI 风控。
- 默认仍是 `AI自动`：保证总有节点、不黑洞；若 ChatGPT 打不开且你有住宅节点，手动切 `AI住宅` / `AI故障转移`。若订阅无住宅节点，两组为空，主组回退 `AI自动`。

在策略组层面用正则筛选，免维护节点名单；机场更新订阅后自动生效。

## 五、分流规则源与 CDN（防回环）
- **远程规则集**（`[filter_remote]`）：`Advertising`（广告拦截）+ `Privacy`（隐私 / 追踪）+ `Hijacking`（运营商反劫持），均 blackmatrix7 QX 版且 `force-policy=广告拦截`（reject）。其余远程列表移除，少拉远程、冷启动更快。
- **本地手写**（`[filter_local]`，最高优先、冷启动也生效）：局域网 / 私网直连 → OpenAI / Claude / GoogleAI / Gemini(网页) / GoogleSearch → YouTube / TikTok / GitHub / Telegram(含官方 IP 段) / 国际媒体 / Spotify → 苹果 / 微软（直连）精确域名 → 各自策略组；末尾 `geoip,cn → 国内服务` + `final → 漏网之鱼` 兜底。
- 为何手写 AI 域名：blackmatrix7 的 `Gemini.list / Google.list` 把 AI Studio / Gemini API / 搜索混在一起，无法实现 GoogleAI / Gemini / GoogleSearch 的拆分，故不引用，改在本地按 `aistudio.google.com / generativelanguage…→GoogleAI`、`gemini.google.com→Gemini`、`www.google.com→GoogleSearch` 精确分流。

**CDN 统一用 `testingcf.jsdelivr.net`（大陆可直连）**，覆盖：订阅解析脚本 `resource_parser_url`、`geo_location_checker`、`[filter_remote]` 规则集、策略组图标。
- 不用 `cdn.jsdelivr.net`：大陆常被 DNS 污染 → 规则 / 解析脚本拉不下来 → 分流失效全压兜底。
- 配置订阅链接也走 jsDelivr（见 README），不用 `raw.githubusercontent.com`：后者大陆直连常被墙，冷启动「无节点 ↔ 拉不到配置」形成回环。
- 仍拉不动可换 `gcore.jsdelivr.net` / `fastly.jsdelivr.net`，或开「资源解析使用代理」。
