# Quantumult X 简约配置 · 设计文档

## 一、需求
1. 订阅解析（机场订阅链接自动解析为节点）。
2. AI 策略组：OpenAI、Gemini、Claude、Google（AI / 搜索）。
   - 每个 AI 平台同时支持「自动选择」与「手动选择具体节点」。
   - 自动选择池排除：不支持 AI 的地区（香港 / 中国大陆 / 回国）与订阅里的垃圾信息节点（流量 / 到期 / 官网等）。
   - 自动节点不可用时由 `url-latency-benchmark` 定时重测自动切换。
3. 业务分流：苹果服务、国内服务、微软服务、GitHub、YouTube、TikTok、国际媒体（主流视频 / 流媒体）。
4. 广告拦截、兜底分流（漏网之鱼）。
5. 策略组不带 emoji，但带统一风格图标。
6. 每个分流尽量排除其不支持的国家 / 地区。

## 二、图标方案（同类风格统一，统一走 jsDelivr CDN）
| 区域 | 图标库 | 说明 |
|---|---|---|
| AI 区（OpenAI / Gemini / Claude / Google） | lobehub/lobe-icons | AI 品牌官方图标，四者风格统一 |
| 功能 / 地区区 | Koolson/Qure（Color） | QuantumultX 经典统一卡片风格 |

- lobehub base：`https://cdn.jsdelivr.net/gh/lobehub/lobe-icons@master/packages/static-png/light/`
  - OpenAI=`openai.png`（官方为黑白标，无 color 变体）、Gemini=`gemini-color.png`、Claude=`claude-color.png`、Google=`google-color.png`
- Koolson base：`https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/`

选型原因：没有任何单一图标库能同时覆盖「AI 专属图标」与「地区 / 广告 / 国内 / 兜底」全场景（Koolson 缺 Claude / Gemini）。因此 AI 区用 lobehub（保证四个 AI 互相一致、辨识度高），其余用 Koolson（互相一致）。两者均走 `cdn.jsdelivr.net`，国内可达性较好且来源统一。

## 三、策略组结构
基础组（被业务组引用）：
- `全局自动`：url-latency-benchmark，全部节点（仅排垃圾）。
- `AI自动`：url-latency-benchmark，AI 友好地区（排 香港 / 回国 / 中国 + 垃圾）。
- 地区组：`香港 / 台湾 / 日本 / 新加坡 / 美国 / 韩国`，各自 url-latency-benchmark + 正则筛选 + 排垃圾。
- `代理`：static 手动总组（全局自动 + 各地区 + 全部节点手动池）。

业务组（static）：
- AI：`OpenAI / Gemini / Claude / Google` = `AI自动` + AI 友好地区组 + `server-tag-regex` 动态列出 AI 友好节点（供手动选）。
- `GitHub / YouTube / 国际媒体`：代理 + 地区组。
- `TikTok`：仅 日本 / 台湾 / 美国 / 新加坡 / 韩国（排除香港 / 大陆）。
- `微软服务 / 苹果服务 / 国内服务`：默认 `direct`，可切代理。
- `广告拦截`：默认 `reject`。
- `漏网之鱼`：兜底，代理优先。

## 四、地区筛选与垃圾排除（server-tag-regex）
统一排除关键词：`剩余|过期|官网|流量|游戏|失联|客服|群组|倍率|网址|测试|expire|distance` 等。
- AI 友好：额外排除 `香港|HK|hong|回国|中国`，包含 `美|日|新|台|韩|英|德`。
- 各地区组：包含本地区关键词，排除垃圾。

在策略组层面用正则筛选节点，免维护节点名单；机场更新订阅后自动生效。

## 五、分流规则源
blackmatrix7/ios_rule_script（QuantumultX 版），走 jsDelivr。
顺序：局域网 → 广告 → AI(OpenAI / Claude / Gemini) → Google → GitHub → YouTube → TikTok → 国际媒体 → 微软 → 苹果 → 国内；本地 `geoip,cn` + `final` 兜底。
