# Quantumult X 简约配置 · 设计文档

## 一、需求
1. 订阅解析（机场订阅链接自动解析为节点）。
2. AI 策略组：OpenAI、Gemini、Claude、Google（AI / 搜索）。
   - 每个 AI 平台同时支持「自动选择」与「手动选择具体节点」。
   - 自动选择池按各平台**实际支持地区**筛选（详见第四节）：OpenAI / Claude 排 香港 + 俄罗斯 + 中国；Gemini 放行香港；Google 仅排中国大陆；并统一排除订阅里的垃圾信息节点（流量 / 到期 / 官网等）。
   - 自动节点不可用时由 `url-latency-benchmark` 定时重测自动切换。
3. 业务分流：苹果服务、国内服务、微软服务、GitHub、YouTube、TikTok、国际媒体（主流视频 / 流媒体）。
4. 广告拦截、兜底分流（漏网之鱼）。
5. 策略组不带 emoji，但带统一风格图标。
6. 每个分流尽量排除其不支持的国家 / 地区。

## 二、图标方案（同类风格统一，统一走 jsDelivr CDN）
| 区域 | 图标库 | 说明 |
|---|---|---|
| AI 区（OpenAI / Gemini / Claude / Google） | lobehub/lobe-icons | AI 品牌官方图标，四者风格统一 |
| 功能 / 地区区 | Koolson/Qure（Color） | QuantumultX 经典统一卡片风格 |

- lobehub base：`https://cdn.jsdelivr.net/npm/@lobehub/icons-static-png/light/`（走 npm 精简包，而非 `gh` 大仓库路径；`gh` 回源 lobe-icons 巨型 monorepo 时 jsDelivr 易超时，国内常加载失败、显示为占位图标）
  - OpenAI=`openai.png`（官方为黑白标，无 color 变体）、Gemini=`gemini-color.png`、Claude=`claude-color.png`、Google=`google-color.png`
- Koolson base：`https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/`

选型原因：没有任何单一图标库能同时覆盖「AI 专属图标」与「地区 / 广告 / 国内 / 兜底」全场景（Koolson 缺 Claude / Gemini）。因此 AI 区用 lobehub（保证四个 AI 互相一致、辨识度高），其余用 Koolson（互相一致）。两者均走 `cdn.jsdelivr.net`，国内可达性较好且来源统一。

## 三、策略组结构
基础组（被业务组引用）：
- `全局自动`：url-latency-benchmark，全部节点（仅排垃圾）。
- `AI自动`（OpenAI / Claude 用）：封 香港 + 俄罗斯 + 中国，仅取 AI 友好地区。
- `Gemini自动`（Gemini 用）：同上但**放行香港**（Gemini 2026/3 起支持港），仅封 俄罗斯 + 中国。
- `Google自动`（Google 搜索用）：仅封 中国大陆 + 垃圾，含香港 / 俄罗斯等全部地区。
- `代理`：static 手动总组（全局自动 + direct + `server-tag-regex` 全部节点手动池）。

> 注：原「地区自动测速组（香港 / 台湾 / 日本 / 新加坡 / 美国 / 韩国）」已移除；业务组改为依赖 `全局自动 / AI自动` 自动测速 + `代理` 手动选择，免维护一堆地区分组。

业务组（static）：
- AI（各平台支持地区不同，分别配自动组 + 手动池）：
  - `OpenAI` / `Claude` = `AI自动` + 手动池（封 港 / 俄 / 中）。
  - `Gemini` = `Gemini自动` + 手动池（放行香港）。
  - `Google` = `Google自动` + 手动池（仅封中国大陆）。
- `GitHub`：代理 + 全局自动 + direct。
- `YouTube / TikTok / 国际媒体`：代理 + 全局自动。
- `微软服务 / 苹果服务 / 国内服务`：默认 `direct`，可切代理。
- `广告拦截`：默认 `reject`。
- `漏网之鱼`：兜底，代理优先。

## 四、各 AI 平台地区差异（2026 查证）与 server-tag-regex
统一排除垃圾：`剩余|过期|官网|流量|游戏|失联|客服|群组|倍率|网址|测试|expire|distance|回国` 等。

各平台可用地区经查证存在真实差异，故**分别筛选**（不再一刀切）：

| 地区 | OpenAI | Claude | Gemini | Google(搜索) |
|---|---|---|---|---|
| 香港 | ❌ | ❌ | ✅(2026/3 起) | ✅ |
| 俄罗斯 | ❌ | ❌ | ❌(制裁) | ✅ |
| 美/日/韩/新/台/英/德/法/加/澳 | ✅ | ✅ | ✅ | ✅ |
| 中国大陆 | ❌ | ❌ | ❌ | ❌ |

- `OpenAI / Claude`（走 `AI自动`）：排除 `香港|港|HK|Hong` + `俄|Russia|莫斯科|RUS` + `中国|大陆|CN`，白名单取 `美|US|日|JP|韩|KR|新|SG|台|TW|英|UK|德|DE|法|FR|加拿大|Canada|澳|AU|荷|NL`。
- `Gemini`（走 `Gemini自动`）：白名单额外放行 `香港|港|HK|Hong`，仍排 俄罗斯 + 中国。
- `Google`（走 `Google自动`）：仅排 中国大陆 + 垃圾，其余全收（含香港 / 俄罗斯）。
- **俄罗斯节点归属**：OpenAI / Claude / Gemini 均封禁 → 俄罗斯节点只会出现在 `Google` 与 `全局自动 / 代理`，不会误入 AI 三家。
- `全局自动 / 代理`：仅排垃圾，覆盖全部节点。

在策略组层面用正则筛选节点，免维护节点名单；机场更新订阅后自动生效。
